3.2. Application¶

class mailingshark.mailingshark.MailingSHARK[source]¶

Main application class. Contains the most important process logic. The main application consists of different steps:

  1. Connects to MongoDB
  2. Finds the project, to which the mailing list belongs
  3. Generates a new mailing list entry, if not existent

4. Finds the correct backend via mailingshark.datacollection.basedatacollector.BaseDataCollector.find_fitting_backend()

5. Downloads the mailboxes using the backend (mailingshark.datacollection.basedatacollector.BaseDataCollector.download_mail_boxes())

  1. Unpacks the files (if necessary) via mailingshark.mailingshark.MailingSHARK._unpack_files()
  2. Parses the messages, by created mailingshark.analyzer.ParsedMessage objects
  3. Stores the messages, by calling mailingshark.mailingshark.MailingSHARK._store_message()
